    Hosts API¶

    The Hosts API manages network hosts (assets) discovered during security assessments. Hosts are the fundamental unit of asset inventory in Faraday and serve as the parent object for services and vulnerabilities.

    All endpoints require authentication via API token (Authorization: Token <token>) or session cookie.

    Base URL: https://localhost:5985

    Host Schema¶

    Property Type Required Description
    _id integer read-only Internal document ID.
    _rev string read-only Document revision identifier.
    command_id integer write-only ID of the command that created this host. Accepted on input but not returned in responses.
    credentials integer read-only Count of credentials associated with this host.
    default_gateway string (nullable) no Default gateway address.
    description string yes Free-text description of the host.
    hostnames array of strings no DNS hostnames associated with the host.
    id integer read-only Primary key. Same as _id.
    importance integer (0--10) no Asset importance rating. 0 = unrated, 10 = critical.
    ip string no IPv4 or IPv6 address.
    mac string no MAC address.
    metadata object read-only Timestamps and user-tracking metadata (create_time, update_time, update_user, etc.).
    name string read-only Display name. Auto-populated by the server.
    os string no Operating system fingerprint.
    owned boolean no Whether the host has been "owned" (compromised) during the assessment.
    owner string read-only Username of the creator.
    services integer read-only Count of services discovered on this host.
    tags array no Tags associated with this host.
    vulns integer read-only Count of vulnerabilities associated with this host.
    type string read-only Always "Host".
    service_summaries array read-only Summary objects for each service (name, port, protocol).
    severity_counts object read-only Vulnerability counts grouped by severity (critical, high, med, low, info, unclassified).
    versions array read-only Detected software versions.

    Required fields for creation: description

    Example Host Object¶

    {
  "_id": 42,
  "_rev": "",
  "command_id": null,
  "default_gateway": [],
  "description": "Primary web server",
  "hostnames": ["web.example.com", "www.example.com"],
  "id": 42,
  "importance": 7,
  "ip": "192.168.1.1",
  "mac": "00:1A:2B:3C:4D:5E",
  "metadata": {
    "create_time": "2026-02-27T10:30:00+00:00",
    "creator": "admin",
    "owner": "admin",
    "update_action": 0,
    "update_time": "2026-02-27T12:15:00+00:00",
    "update_user": "admin"
  },
  "name": "192.168.1.1",
  "os": "Linux 5.15",
  "owned": false,
  "owner": "admin",
  "services": 3,
  "vulns": 12,
  "type": "Host",
  "service_summaries": [
    "ssh (22/tcp)",
    "http (80/tcp)",
    "https (443/tcp)"
  ],
  "severity_counts": {
    "critical": 1,
    "high": 3,
    "med": 4,
    "low": 2,
    "info": 2,
    "unclassified": 0
  },
  "versions": [],
  "workspace_name": "pentest_q1"
}

    HostCount Schema¶

    Returned by the countVulns endpoints.

    Property Type Description
    host_id integer Host primary key.
    critical integer Count of critical-severity vulnerabilities.
    high integer Count of high-severity vulnerabilities.
    med integer Count of medium-severity vulnerabilities.
    low integer Count of low-severity vulnerabilities.
    info integer Count of informational-severity vulnerabilities.
    unclassified integer Count of unclassified vulnerabilities.
    total integer Total vulnerability count.

    Workspaced Endpoints¶

    All workspaced endpoints operate under the path prefix /_api/v3/ws/{workspace_name}/hosts. Replace {workspace_name} with the target workspace slug.

    List Hosts¶

    Retrieve a paginated list of all hosts in a workspace.
    Method GET
    Path /_api/v3/ws/{workspace_name}/hosts
    Auth Required

    Query Parameters:

    Parameter Type Default Description
    page integer 0 Page number (zero-indexed).
    page_size integer 20 Number of results per page.
    sort string ip Field to sort by.
    sort_dir string asc Sort direction (asc or desc).

    Response: 200 OK

    {
  "count": 150,
  "data": [
    { "...host object..." }
  ]
}

    Example:

    curl -s -X GET \
  "https://localhost:5985/_api/v3/ws/pentest_q1/hosts?page=0&page_size=10" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -k

    Error Codes:

    Code Reason
    401 Missing or invalid authentication.
    403 User lacks permission for this workspace.
    404 Workspace not found.

    Create Host¶

    Create a new host in the specified workspace.
    Method POST
    Path /_api/v3/ws/{workspace_name}/hosts
    Auth Required

    Request Body:

    {
  "ip": "192.168.1.1",
  "description": "Web server",
  "hostnames": ["web.example.com"],
  "os": "Linux",
  "mac": "00:1A:2B:3C:4D:5E",
  "importance": 5,
  "owned": false,
  "default_gateway": []
}

    Required fields: description

    Response: 201 Created

    Returns the newly created host object.

    Example:

    curl -s -X POST \
  "https://localhost:5985/_api/v3/ws/pentest_q1/hosts" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -k \
  -d '{
    "ip": "192.168.1.1",
    "description": "Web server",
    "hostnames": ["web.example.com"],
    "os": "Linux"
  }'

    Error Codes:

    Code Reason
    400 Validation error (e.g., missing description).
    401 Missing or invalid authentication.
    403 User lacks permission for this workspace.
    404 Workspace not found.
    409 Host with this IP already exists in the workspace.

    Get Host¶

    Retrieve a single host by its ID.
    Method GET
    Path /_api/v3/ws/{workspace_name}/hosts/{object_id}
    Auth Required

    Path Parameters:

    Parameter Type Description
    workspace_name string Workspace slug.
    object_id integer Host ID.

    Response: 200 OK

    Returns the full host object.

    Example:

    curl -s -X GET \
  "https://localhost:5985/_api/v3/ws/pentest_q1/hosts/42" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -k

    Error Codes:

    Code Reason
    401 Missing or invalid authentication.
    403 User lacks permission for this workspace.
    404 Host or workspace not found.

    Update Host (Full)¶

    Replace all mutable fields of a host. Fields not included in the request body are reset to their defaults.
    Method PUT
    Path /_api/v3/ws/{workspace_name}/hosts/{object_id}
    Auth Required

    Request Body:

    {
  "ip": "192.168.1.1",
  "description": "Updated web server",
  "hostnames": ["web.example.com", "api.example.com"],
  "os": "Linux 5.15",
  "mac": "00:1A:2B:3C:4D:5E",
  "importance": 8,
  "owned": true,
  "default_gateway": ["192.168.1.254"]
}

    Response: 200 OK

    Returns the updated host object.

    Example:

    curl -s -X PUT \
  "https://localhost:5985/_api/v3/ws/pentest_q1/hosts/42" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -k \
  -d '{
    "ip": "192.168.1.1",
    "description": "Updated web server",
    "hostnames": ["web.example.com", "api.example.com"],
    "os": "Linux 5.15",
    "importance": 8,
    "owned": true
  }'

    Error Codes:

    Code Reason
    400 Validation error.
    401 Missing or invalid authentication.
    403 User lacks permission for this workspace.
    404 Host or workspace not found.
    409 IP conflict with another host in the workspace.

    Update Host (Partial)¶

    Update only the specified fields of a host. Omitted fields remain unchanged.
    Method PATCH
    Path /_api/v3/ws/{workspace_name}/hosts/{object_id}
    Auth Required

    Request Body:

    {
  "importance": 10,
  "owned": true
}

    Response: 200 OK

    Returns the updated host object.

    Example:

    curl -s -X PATCH \
  "https://localhost:5985/_api/v3/ws/pentest_q1/hosts/42" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -k \
  -d '{"importance": 10, "owned": true}'

    Error Codes:

    Code Reason
    400 Validation error.
    401 Missing or invalid authentication.
    403 User lacks permission for this workspace.
    404 Host or workspace not found.

    Delete Host¶

    Delete a single host and all of its associated services and vulnerabilities.
    Method DELETE
    Path /_api/v3/ws/{workspace_name}/hosts/{object_id}
    Auth Required

    Path Parameters:

    Parameter Type Description
    workspace_name string Workspace slug.
    object_id integer Host ID.

    Response: 204 No Content

    No response body.

    Example:

    curl -s -X DELETE \
  "https://localhost:5985/_api/v3/ws/pentest_q1/hosts/42" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -k

    Error Codes:

    Code Reason
    401 Missing or invalid authentication.
    403 User lacks permission for this workspace.
    404 Host or workspace not found.

    Bulk Delete Hosts¶

    Delete multiple hosts in a single request.
    Method DELETE
    Path /_api/v3/ws/{workspace_name}/hosts
    Auth Required

    Request Body:

    {
  "ids": [1, 2, 3]
}

    Response: 204 No Content

    No response body on success.

    Example:

    curl -s -X DELETE \
  "https://localhost:5985/_api/v3/ws/pentest_q1/hosts" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -k \
  -d '{"ids": [1, 2, 3]}'

    Error Codes:

    Code Reason
    400 Invalid request body or empty ids array.
    401 Missing or invalid authentication.
    403 User lacks permission for this workspace.
    404 Workspace not found.

    Bulk Update Hosts¶

    Update multiple hosts with the same set of field values.
    Method PATCH
    Path /_api/v3/ws/{workspace_name}/hosts
    Auth Required

    Request Body:

    {
  "ids": [1, 2],
  "update": {
    "owned": true,
    "importance": 9
  }
}

    Response: 200 OK

    Returns a count of updated objects.

    {
  "updated": 2
}

    Example:

    curl -s -X PATCH \
  "https://localhost:5985/_api/v3/ws/pentest_q1/hosts" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -k \
  -d '{"ids": [1, 2], "update": {"owned": true}}'

    Error Codes:

    Code Reason
    400 Invalid request body, empty ids, or invalid fields in update.
    401 Missing or invalid authentication.
    403 User lacks permission for this workspace.
    404 Workspace not found.

    Count Hosts¶

    Return the total number of hosts in a workspace, optionally grouped by a field.
    Method GET
    Path /_api/v3/ws/{workspace_name}/hosts/count
    Auth Required

    Query Parameters:

    Parameter Type Default Description
    group_by string (none) Field to group counts by (e.g., os, owned, importance).

    Response: 200 OK

    Without group_by:

    {
  "count": 150
}

    With group_by=os:

    {
  "groups": [
    {"count": 80, "os": "Linux"},
    {"count": 45, "os": "Windows"},
    {"count": 25, "os": ""}
  ]
}

    Example:

    curl -s -X GET \
  "https://localhost:5985/_api/v3/ws/pentest_q1/hosts/count?group_by=os" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -k

    Error Codes:

    Code Reason
    401 Missing or invalid authentication.
    403 User lacks permission for this workspace.
    404 Workspace not found.

    Filter Hosts¶

    Retrieve hosts matching a filter expression.
    Method GET
    Path /_api/v3/ws/{workspace_name}/hosts/filter
    Auth Required

    Query Parameters:

    Parameter Type Description
    q string (JSON) Filter expression. See Filter Syntax below.
    page integer Page number (zero-indexed).
    page_size integer Results per page.

    Response: 200 OK

    Same paginated format as List Hosts.

    Example:

    curl -s -X GET \
  "https://localhost:5985/_api/v3/ws/pentest_q1/hosts/filter?q=%7B%22filters%22%3A%5B%7B%22name%22%3A%22os%22%2C%22op%22%3A%22like%22%2C%22val%22%3A%22%25Linux%25%22%7D%5D%7D" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -k

    Decoded q parameter:

    {
  "filters": [
    {"name": "os", "op": "like", "val": "%Linux%"}
  ]
}

    Error Codes:

    Code Reason
    400 Malformed filter expression.
    401 Missing or invalid authentication.
    403 User lacks permission for this workspace.
    404 Workspace not found.

    Filter Syntax¶

    Filters use a JSON structure with an array of filter objects:

    {
  "filters": [
    {"name": "<field>", "op": "<operator>", "val": "<value>"}
  ],
  "conjunction": "AND"
}

    Supported operators: eq, ne, lt, le, gt, ge, like, ilike, in, not_in, is_null, is_not_null

    Global (Non-Workspaced) Endpoints¶

    Important: These global host endpoints are not declared in the published 5.10.0 OpenAPI specification (faraday_swagger.json). They exist in the codebase spec for version 5.17.0+. The paths and behaviors described below are derived from the codebase and may not be available in all Faraday deployments.

    Global endpoints operate under the path prefix /_api/v3/hosts and provide read-only cross-workspace access (with the exception of bulk delete and bulk update). No {workspace_name} is required.

    List All Hosts¶

    Retrieve a paginated list of hosts across all accessible workspaces.
    Method GET
    Path /_api/v3/hosts
    Auth Required

    Query Parameters:

    Parameter Type Default Description
    page integer 0 Page number (zero-indexed).
    page_size integer 20 Number of results per page.
    sort string ip Field to sort by.
    sort_dir string asc Sort direction (asc or desc).

    Response: 200 OK

    {
  "count": 500,
  "data": [
    { "...host object with workspace_name..." }
  ]
}

    Each host object includes workspace_name to identify which workspace it belongs to.

    Example:

    curl -s -X GET \
  "https://localhost:5985/_api/v3/hosts?page=0&page_size=50" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -k

    Error Codes:

    Code Reason
    401 Missing or invalid authentication.

    Get Host (Global)¶

    Retrieve a single host by ID without specifying a workspace.
    Method GET
    Path /_api/v3/hosts/{object_id}
    Auth Required

    Response: 200 OK

    Returns the full host object including workspace_name.

    Example:

    curl -s -X GET \
  "https://localhost:5985/_api/v3/hosts/42" \
  -H "Authorization: Token YOUR_API_TOKEN" \
  -k

    Error Codes:

    Code Reason
    401 Missing or invalid authentication.
    403 User lacks permission for the host's workspace.
    404 Host not found.

  -H "Authorization: Token YOUR_API_TOKEN" \
  -k

    Error Codes:

    Code Reason
    400 Malformed filter expression.
    401 Missing or invalid authentication.

    Deprecated Endpoints¶

    Deprecated Path Replacement

    The legacy bulk_delete path is intentionally blocked via SKIP_RULES. Any requests to it will return 404. Use the standard DELETE on the collection endpoint with an ids array in the request body instead.

    Common Error Codes¶

    HTTP Status Meaning
    200 Success. Response body contains the requested data.
    201 Created. New resource was successfully created.
    204 No Content. Resource was deleted successfully.
    400 Bad Request. Malformed JSON, missing required fields, or invalid filter expression.
    401 Unauthorized. Authentication token is missing, expired, or invalid.
    403 Forbidden. Authenticated user does not have permission for the requested workspace or resource.
    404 Not Found. The workspace, host, or endpoint does not exist.
    409 Conflict. A host with the same IP already exists in the workspace.
    500 Internal Server Error. Unexpected server-side failure.

    Authentication¶

    All requests must include one of the following:

    API Token (recommended):

    Authorization: Token <your_api_token>

    Obtain a token from the Faraday web UI under Settings > API Tokens, or via:

    curl -s -X POST \
  "https://localhost:5985/_api/v3/token" \
  -H "Content-Type: application/json" \
  -k \
  -d '{"username": "faraday", "password": "changeme"}'

    Session Cookie:

    Authenticate via the login endpoint and pass the session cookie with subsequent requests.

    Notes¶

    • All curl examples include -k to skip TLS certificate verification for local development. Remove this flag in production environments with valid certificates.
    • Pagination is zero-indexed: page=0 returns the first page.
    • The importance field accepts values from 0 (unrated) to 10 (critical asset).
    • Host IPs must be unique within a workspace. Attempting to create a duplicate returns 409 Conflict.
    • Deleting a host cascades to its services and vulnerabilities.
