    Services API¶

    API Version: v3 Base URL: https://<faraday-server>/_api Authentication: Cookie-based session or API token (Authorization: Token <token>) Content-Type: application/json

    Services represent network services (ports) discovered on hosts within a workspace. Each service belongs to a single host and can have zero or more vulnerabilities associated with it.

    Service Schema¶

    Field Type Required Description
    _id integer -- Internal object identifier (read-only).
    _rev string -- Revision hash (read-only).
    id integer -- Service ID (read-only, same as _id).
    name string No Service name (e.g. "http", "ssh", "ftp").
    description string No Free-text description of the service.
    port integer Yes Port number (e.g. 80, 443, 22).
    ports string Yes Port number as a string (e.g. "80", "443").
    protocol string Yes Transport protocol (e.g. "tcp", "udp").
    status string No Port status (e.g. "open", "closed", "filtered").
    version string No Service version banner (e.g. "Apache 2.4.41", "OpenSSH 8.2").
    owned boolean No Whether the service has been marked as owned. Default: false.
    owner string No Username of the object owner (read-only, set by server).
    host_id integer No ID of the parent host (read-only).
    parent integer No Parent host ID. Used when creating a service to attach it to a host.
    parent_name string No Hostname of the parent host (read-only).
    command_id integer No ID of the command that created this service (read-only).
    vulns integer No Count of vulnerabilities associated with this service (read-only).
    metadata object No Timestamps and metadata (read-only). Contains create_time, update_time, update_user, update_action, creator, owner, command_id.
    summary string No Service summary text (read-only).
    credentials integer No Count of credentials associated with this service (read-only).
    tags array No Tags associated with this service.
    type string No Object type identifier (read-only, typically "Service").

    Example Service Object¶

    {
  "_id": 42,
  "_rev": "",
  "id": 42,
  "name": "http",
  "description": "Web server",
  "port": 80,
  "ports": "80",
  "protocol": "tcp",
  "status": "open",
  "version": "Apache 2.4.41",
  "owned": false,
  "owner": "faraday",
  "host_id": 1,
  "parent": 1,
  "parent_name": "192.168.1.10",
  "command_id": null,
  "vulns": 3,
  "metadata": {
    "create_time": "2026-02-27T10:30:00+00:00",
    "update_time": "2026-02-27T10:30:00+00:00",
    "update_user": "faraday",
    "update_action": 0,
    "creator": "faraday",
    "owner": "faraday",
    "command_id": null
  },
  "summary": {},
  "type": "Service",
  "workspace_name": "pentest"
}

    Workspaced Endpoints¶

    These endpoints operate within a specific workspace. Replace {workspace_name} with the target workspace name.

    Base path: /_api/v3/ws/{workspace_name}/services

    1. List Services¶

    Retrieve a paginated list of all services in a workspace.
    Method GET
    Path /_api/v3/ws/{workspace_name}/services
    Auth Session cookie or Authorization: Token <token>

    Query Parameters¶

    Parameter Type Default Description
    page integer 0 Page number (0-indexed).
    page_size integer 20 Number of results per page.
    sort string -- Field to sort by (e.g. port, name, status).
    sort_dir string asc Sort direction: asc or desc.

    Response: 200 OK¶

    2. Create Service¶

    Create a new service attached to an existing host in the workspace.
    Method POST
    Path /_api/v3/ws/{workspace_name}/services
    Auth Session cookie or Authorization: Token <token>

    Request Body¶

    Field Type Required Description
    name string No Service name (e.g. "http").
    port integer Yes Port number.
    ports string Yes Port number as string.
    protocol string Yes Transport protocol (e.g. "tcp").
    status string No Port status (e.g. "open").
    version string No Service version string.
    description string No Free-text description.
    owned boolean No Mark as owned. Default: false.
    parent integer No Host ID to attach the service to.

    Request Example¶

    {
  "name": "http",
  "port": 80,
  "ports": "80",
  "protocol": "tcp",
  "status": "open",
  "version": "Apache 2.4.41",
  "description": "Main web server",
  "parent": 1
}

    Response: 201 Created¶

    Returns the full service object with server-assigned fields (_id, id, metadata, owner, etc.).

    {
  "_id": 44,
  "_rev": "",
  "id": 44,
  "name": "http",
  "description": "Main web server",
  "port": 80,
  "ports": "80",
  "protocol": "tcp",
  "status": "open",
  "version": "Apache 2.4.41",
  "owned": false,
  "owner": "faraday",
  "host_id": 1,
  "parent": 1,
  "parent_name": "192.168.1.10",
  "command_id": null,
  "vulns": 0,
  "metadata": {
    "create_time": "2026-02-27T11:00:00+00:00",
    "update_time": "2026-02-27T11:00:00+00:00",
    "update_user": "faraday",
    "update_action": 0,
    "creator": "faraday",
    "owner": "faraday",
    "command_id": null
  },
  "type": "Service",
  "workspace_name": "pentest"
}

    Example¶

    curl -s -X POST -H "Authorization: Token $TOKEN" \
  -H "Content-Type: application/json" \
  "https://localhost:5985/_api/v3/ws/pentest/services" \
  -d '{
    "name": "http",
    "port": 80,
    "ports": "80",
    "protocol": "tcp",
    "status": "open",
    "version": "Apache 2.4.41",
    "parent": 1
  }'

    Error Responses¶

    Code Condition
    400 Missing required fields (port, ports, protocol), or invalid field values.
    401 Not authenticated.
    404 Workspace not found, or parent host does not exist.

    3. Get Service¶

    Retrieve a single service by its ID.
    Method GET
    Path /_api/v3/ws/{workspace_name}/services/{object_id}
    Auth Session cookie or Authorization: Token <token>

    Path Parameters¶

    Parameter Type Description
    workspace_name string Target workspace name.
    object_id integer Service ID.

    Response: 200 OK¶

    Returns the full service object (same schema as the create response).

    Example¶

    curl -s -H "Authorization: Token $TOKEN" \
  "https://localhost:5985/_api/v3/ws/pentest/services/42"

    Error Responses¶

    Code Condition
    401 Not authenticated.
    404 Workspace or service not found.

    4. Update Service (Full)¶

    Replace all fields of a service. All mutable fields must be provided; omitted fields will be reset to defaults.
    Method PUT
    Path /_api/v3/ws/{workspace_name}/services/{object_id}
    Auth Session cookie or Authorization: Token <token>

    Request Body¶

    Same schema as Create Service. All required fields must be present.

    Request Example¶

    {
  "name": "https",
  "port": 443,
  "ports": "443",
  "protocol": "tcp",
  "status": "open",
  "version": "nginx 1.21",
  "description": "Updated to HTTPS",
  "owned": false,
  "parent": 1
}

    Response: 200 OK¶

    Returns the updated service object.

    Example¶

    curl -s -X PUT -H "Authorization: Token $TOKEN" \
  -H "Content-Type: application/json" \
  "https://localhost:5985/_api/v3/ws/pentest/services/42" \
  -d '{
    "name": "https",
    "port": 443,
    "ports": "443",
    "protocol": "tcp",
    "status": "open",
    "version": "nginx 1.21",
    "description": "Updated to HTTPS",
    "owned": false,
    "parent": 1
  }'

    Error Responses¶

    Code Condition
    400 Missing required fields or invalid values.
    401 Not authenticated.
    404 Workspace or service not found.

    5. Update Service (Partial)¶

    Update one or more fields of a service without replacing the entire object.
    Method PATCH
    Path /_api/v3/ws/{workspace_name}/services/{object_id}
    Auth Session cookie or Authorization: Token <token>

    Request Body¶

    Only include the fields you want to change.

    Request Example¶

    {
  "status": "closed",
  "version": "Apache 2.4.52"
}

    Response: 200 OK¶

    Returns the updated service object.

    Example¶

    curl -s -X PATCH -H "Authorization: Token $TOKEN" \
  -H "Content-Type: application/json" \
  "https://localhost:5985/_api/v3/ws/pentest/services/42" \
  -d '{
    "status": "closed",
    "version": "Apache 2.4.52"
  }'

    Error Responses¶

    Code Condition
    400 Invalid field values.
    401 Not authenticated.
    404 Workspace or service not found.

    6. Delete Service¶

    Delete a single service by its ID. This also removes the association with any linked vulnerabilities.
    Method DELETE
    Path /_api/v3/ws/{workspace_name}/services/{object_id}
    Auth Session cookie or Authorization: Token <token>

    Response: 204 No Content¶

    Empty response body on success.

    Example¶

    curl -s -X DELETE -H "Authorization: Token $TOKEN" \
  "https://localhost:5985/_api/v3/ws/pentest/services/42"

    Error Responses¶

    Code Condition
    401 Not authenticated.
    404 Workspace or service not found.

    7. Bulk Delete Services¶

    Delete multiple services in a single request.
    Method DELETE
    Path /_api/v3/ws/{workspace_name}/services
    Auth Session cookie or Authorization: Token <token>

    Request Body¶

    {
  "ids": [42, 43, 44]
}
    Field Type Required Description
    ids array of integers Yes List of service IDs to delete.

    Response: 200 OK¶

    Returns a confirmation with the count of deleted services.

    {
  "deleted": 3
}

    Example¶

    curl -s -X DELETE -H "Authorization: Token $TOKEN" \
  -H "Content-Type: application/json" \
  "https://localhost:5985/_api/v3/ws/pentest/services" \
  -d '{"ids": [42, 43, 44]}'

    Error Responses¶

    Code Condition
    400 Missing or empty ids array.
    401 Not authenticated.
    404 Workspace not found.

    8. Bulk Update Services¶

    Update fields on multiple services at once.
    Method PATCH
    Path /_api/v3/ws/{workspace_name}/services
    Auth Session cookie or Authorization: Token <token>

    Request Body¶

    {
  "ids": [42, 43],
  "update": {
    "owned": true,
    "status": "open"
  }
}
    Field Type Required Description
    ids array of integers Yes List of service IDs to update.
    update object Yes Key-value pairs of fields to update.

    Response: 200 OK¶

    Returns a confirmation with the count of updated services.

    {
  "updated": 2
}

    Example¶

    curl -s -X PATCH -H "Authorization: Token $TOKEN" \
  -H "Content-Type: application/json" \
  "https://localhost:5985/_api/v3/ws/pentest/services" \
  -d '{
    "ids": [42, 43],
    "update": {
      "owned": true,
      "status": "open"
    }
  }'

    Error Responses¶

    Code Condition
    400 Missing ids or update, or invalid field values.
    401 Not authenticated.
    404 Workspace not found.

    10. Filter Services¶

    Retrieve services matching a filter expression. Supports the same query parameter filters available throughout the Faraday API.
    Method GET
    Path /_api/v3/ws/{workspace_name}/services/filter
    Auth Session cookie or Authorization: Token <token>

    Query Parameters¶

    Parameter Type Description
    q string (JSON) Filter query in Faraday filter syntax.
    page integer Page number (0-indexed).
    page_size integer Items per page.
    sort string Field to sort by.
    sort_dir string Sort direction (asc or desc).

    Filter Syntax¶

    Filters are passed as a JSON-encoded q parameter. The general structure is:

    q={"filters": [{"name": "<field>", "op": "<operator>", "val": "<value>"}]}

    Supported operators: eq, neq, gt, gte, lt, lte, like, ilike, in, not_in, is_null, is_not_null

    Response: 200 OK¶

    Same paginated response format as List Services.

    Examples¶

    Filter by port:

    curl -s -G -H "Authorization: Token $TOKEN" \
  "https://localhost:5985/_api/v3/ws/pentest/services/filter" \
  --data-urlencode 'q={"filters": [{"name": "port", "op": "eq", "val": 80}]}'

    Filter by status "open" and protocol "tcp":

    curl -s -G -H "Authorization: Token $TOKEN" \
  "https://localhost:5985/_api/v3/ws/pentest/services/filter" \
  --data-urlencode 'q={"filters": [{"name": "status", "op": "eq", "val": "open"}, {"name": "protocol", "op": "eq", "val": "tcp"}]}'

    Filter services with vulnerabilities:

    curl -s -G -H "Authorization: Token $TOKEN" \
  "https://localhost:5985/_api/v3/ws/pentest/services/filter" \
  --data-urlencode 'q={"filters": [{"name": "vulns", "op": "gt", "val": 0}]}'

    Error Responses¶

    Code Condition
    400 Malformed filter query.
    401 Not authenticated.
    404 Workspace not found.

    Global Endpoints¶

    Important: These global service endpoints are not declared in the published 5.10.0 OpenAPI specification (faraday_swagger.json). They exist in the codebase spec for version 5.17.0+. The paths and behaviors described below are derived from the codebase and may not be available in all Faraday deployments.

    Global endpoints provide read-only (and bulk operation) access to services across all workspaces the authenticated user has access to. These are useful for dashboards and cross-workspace reporting.

    Base path: /_api/v3/services

    1. List All Services¶

    Retrieve a paginated list of services across all accessible workspaces.
    Method GET
    Path /_api/v3/services
    Auth Session cookie or Authorization: Token <token>

    Query Parameters¶

    Same pagination and sorting parameters as List Services (page, page_size, sort, sort_dir).

    Response: 200 OK¶

    Same paginated response format as List Services. Each service object includes workspace_name to identify its workspace.

    Example¶

    curl -s -H "Authorization: Token $TOKEN" \
  "https://localhost:5985/_api/v3/services?page=0&page_size=50"

    Error Responses¶

    Code Condition
    401 Not authenticated.

    2. Get Service by ID¶

    Retrieve a single service by its ID without specifying a workspace.
    Method GET
    Path /_api/v3/services/{object_id}
    Auth Session cookie or Authorization: Token <token>

    Path Parameters¶

    Parameter Type Description
    object_id integer Service ID.

    Response: 200 OK¶

    Returns the full service object including workspace_name.

    Example¶

    curl -s -H "Authorization: Token $TOKEN" \
  "https://localhost:5985/_api/v3/services/42"

    Error Responses¶

    Code Condition
    401 Not authenticated.
    404 Service not found or user lacks access to its workspace.

    3. Filter Services Globally¶

    Filter services across all accessible workspaces.
    Method GET
    Path /_api/v3/services/filter
    Auth Session cookie or Authorization: Token <token>

    Query Parameters¶

    Same filter parameters as Filter Services (workspaced).

    Response: 200 OK¶

    Same paginated response format as List Services.

    Example¶

    curl -s -G -H "Authorization: Token $TOKEN" \
  "https://localhost:5985/_api/v3/services/filter" \
  --data-urlencode 'q={"filters": [{"name": "port", "op": "eq", "val": 443}]}'

    Error Responses¶

    Code Condition
    400 Malformed filter query.
    401 Not authenticated.

    Common Error Responses¶

    All endpoints may return the following standard error responses:

    400 Bad Request¶

    Returned when the request body is malformed, required fields are missing, or field values are invalid.

    {
  "messages": ["Missing required field: port"]
}

    401 Unauthorized¶

    Returned when the request lacks valid authentication credentials.

    {
  "messages": ["Unauthorized"]
}

    404 Not Found¶

    Returned when the workspace, service, or parent host does not exist.

    {
  "messages": ["Not Found"]
}

    Usage Notes¶

    Relationship to Hosts¶

    • Every service must belong to a host. When creating a service, use the parent field to specify the host ID.
    • The host_id and parent fields both reference the parent host. host_id is read-only and set by the server; parent is used in write operations.
    • Deleting a host will cascade-delete all its services.

    Relationship to Vulnerabilities¶

    • Vulnerabilities can be associated with a service via the vulnerability's service_id field.
    • The vulns field on a service object is a read-only count of associated vulnerabilities.
    • Deleting a service does not automatically delete its vulnerabilities; they become unlinked.

    Port and Ports Fields¶

    • Both port (integer) and ports (string) are required when creating or fully updating a service. They should contain the same value in their respective types.
    • The ports field exists for historical compatibility and should match the port value as a string.

    Duplicate Services¶

    • Faraday identifies services by the combination of host_id + port + protocol within a workspace. Attempting to create a duplicate may result in an update of the existing service rather than an error, depending on the server configuration and ingestion method.

    Filtering by Host¶

    To retrieve all services for a specific host, use the filter endpoint:

    curl -s -G -H "Authorization: Token $TOKEN" \
  "https://localhost:5985/_api/v3/ws/pentest/services/filter" \
  --data-urlencode 'q={"filters": [{"name": "host_id", "op": "eq", "val": 1}]}'

    Alternatively, services for a host can also be retrieved via the Hosts API:

    curl -s -H "Authorization: Token $TOKEN" \
  "https://localhost:5985/_api/v3/ws/pentest/hosts/1/services"
